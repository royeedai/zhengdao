# 系统/作品设置拆分与工作区统一收口 Mission

## 1. 交付基线摘要

- **宿主项目 / 系统**：证道 Electron 桌面端创作软件
- **当前交付主题**：系统/作品设置拆分与工作区统一收口
- **当前交付目标**：把系统级设置和作品级设置重新分层，补齐系统级题材模板库与系统默认日更目标，修正顶部标题栏/右侧面板/底部创世沙盘交互，并把剩余旧样式页面统一到新主题变量。
- **成功标准**：
  - 系统设置承接外观、题材模板、日更默认、AI 全局账号、账号与云同步、快捷键、备份与迁移、更新与关于
  - 作品设置只保留题材应用、作品日更目标和 AI 作品能力入口
  - 题材模板进入系统级持久化模板库，支持新建、复制、编辑和默认模板
  - 新建作品第二步读取系统模板库，并覆盖“有默认 / 无默认 / 无模板”三种状态
  - 标题栏可清晰区分系统级与作品级入口，且整条标题栏背景支持双击最大化/还原
  - 右侧灵感 tab 显示数量，创世沙盘首节点可见且支持横纵拖拽
  - 总览、风格分析、文本分析、导出、一致性检查、角色对比等旧样式页面接入当前主题变量
- **项目模式**：change
- **当前治理档位**：P1
- **当前基线 ID**：CR-20260423-203543-settings-workspace-polish

## 2. 范围边界

### 范围内

- `genre_templates` 持久化表、系统默认题材模板与系统默认日更目标
- `project_config.daily_goal_mode` 与作品日更目标继承模型
- preload / IPC / renderer store 对应接口
- `AppSettingsModal`、`ProjectSettingsModal`、`NewBookWizard`、`TopBar`、`BookshelfPage`
- 右侧灵感数量 badge 与共享 note store
- 创世沙盘交互和旧样式页面主题统一

### 范围外

- AI provider 底层路由、全局账号 schema、云同步底层语义
- 作品模板 retroactive 同步
- 发布流程、打包配置、原生模块链路

## 3. 已确认约束与关键决策

- 系统级题材模板编辑只影响后续新建作品和后续手动套用，不反向改已有作品。
- 日更目标采用“系统默认 + 作品可切自定义”的继承模型。
- 创世沙盘节点支持横向改章节、纵向改爽度。
- seed 预设允许编辑但不允许删除；复制/新建出来的模板允许删除。
- `AiSettingsModal` 保留，但不再作为作品设置页签的一部分。
