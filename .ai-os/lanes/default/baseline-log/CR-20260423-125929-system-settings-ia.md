# CR-20260423-125929-system-settings-ia

## 背景

最近多轮顶栏、更新入口和账号入口调整后，主题能力仍存在于 `ui-store`，但主要入口留在工作区标题栏主题/账号菜单中；书架页和“应用设置 / 关于”没有外观设置，导致用户认为主题设置消失。

## 用户确认

- 2026-04-23：用户确认执行“系统设置入口收口”方案。
- 本轮范围为：外观主题、Google 账号 / 云同步、更新 / 关于统一进入应用设置。
- 快捷键、备份与迁移暂不迁移，避免把入口修复扩展成高风险数据操作重组。

## 变更范围

### 范围内

- `AppSettingsModal` 改为系统级应用设置，提供“外观 / 账号与云同步 / 更新与关于”页签。
- 主题选择移入应用设置，继续复用 `write-ui-theme` 本地存储与现有 `THEME_IDS`。
- Google 登录、退出、同步开关、手动上传与云端文件列表移入应用设置账号页。
- 工作区标题栏移除主题菜单、账号菜单中的主题列表和独立登录按钮；云图标仅保留状态提示。
- 更新自动弹窗直接进入“更新与关于”页。

### 范围外

- 不迁移项目设置中的快捷键、备份与迁移。
- 不重构主题 token、配色体系或全站视觉。
- 不改变 Google Drive 同步 IPC、数据库 schema 或更新服务主进程逻辑。

## 风险与 Guard

- 风险：`system` 主题被错误写成 `data-theme="system"`，导致 CSS token 不匹配。
- 风险：更新自动弹窗默认进入外观页，影响用户安装新版本路径。
- 风险：标题栏清理后用户无法进入 Google 登录 / 退出流程。

## 验证要求

- `npm test`
- `npm run build`
- 代码审阅确认书架页与工作区均有应用设置入口，且标题栏不再暴露主题/登录设置入口。
