# v1.3 可信日更工作台 Mission

## 1. 交付基线摘要

- **宿主项目 / 系统**：证道 Electron 桌面端创作软件
- **当前交付主题**：v1.3 可信日更工作台
- **当前交付目标**：让中文长篇 / 网文作者每天打开作品后，可以稳定完成选章节、写正文、看进度、确认保存恢复、检查风险、准备发布稿。
- **成功标准**：
  - 工作区有轻量今日状态区，展示今日字数、目标差额、连续日更、当前章节保存状态、最近快照、最近备份和发布准备入口。
  - 本章审稿台输出固定结构报告，覆盖剧情推进、角色一致性、伏笔回收、毒点风险、节奏问题和可执行修改建议。
  - 发布前检查包支持当前章节 / 全书检查，覆盖敏感词、空标题、空正文、字数异常和纯文本预览。
  - 发布准备不接入任何平台 API，不新增平台账号，不改变 Google Drive 同步语义。
  - AI 产生的正文替换、角色、设定、伏笔和剧情节点仍必须进入草稿确认流，不自动写入。
- **项目模式**：change
- **当前治理档位**：P1
- **当前基线 ID**：CR-20260423-v13-daily-workbench

## 2. 范围边界

### 范围内

- 工作区今日状态区
- 保存 / 快照 / 备份状态可见性
- 本章审稿台固定报告
- 审稿后的资产建议确认入口
- 发布前检查包、复制发布稿和 TXT / DOCX / Markdown 导出入口
- 新增纯逻辑测试和项目原生验证

### 范围外

- 拼字社群、战队、排行榜
- 移动端或多端编辑
- 平台 API 发布、账号接入、收益 / 读者数据同步
- 整本自动生成、自动批量改正文
- 专业出版排版、打印 PDF 出版格式重构
- SQLite 核心作品 schema 重构

## 3. 已确认约束与关键决策

- 证道差异化保持为本地优先、长篇资产、可控 AI、网文日更闭环。
- 发布前检查保持平台中立，只做复制 / 导出 / 检查。
- 审稿台复用现有 AI 账号、能力卡、上下文和草稿篮。
- 资产建议只进入确认流，不自动创建角色、设定、伏笔或剧情节点。
- Windows x64 是首要验证平台；本轮不改变打包和更新发布链路。
