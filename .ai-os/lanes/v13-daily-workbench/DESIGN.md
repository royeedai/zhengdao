# v1.3 可信日更工作台 Design

## 1. 设计目标

- **产品目标**：作者进入作品后第一眼知道今天写作进度、正文是否保存、是否有可恢复版本、是否已外部备份、是否可发布。
- **交互目标**：把风险和发布准备放到工作区轻量入口，不打断写作，不新增平台依赖。
- **AI 目标**：把 AI 从泛聊天收口为可验证的本章审稿报告和可确认资产建议。

## 2. 关键页面与交互

| 页面 / 入口 | 改动目标 | 保持不变 |
|---|---|---|
| 工作区今日状态区 | 展示今日字数、目标差额、连续日更、保存、快照、备份和发布准备 | 编辑器、左右栏和底部沙盘主布局 |
| 本章审稿台 | 输出固定结构报告并提供资产建议入口 | AI provider、全局账号、作品 AI 档案 |
| 发布前检查包 | 当前章节 / 全书检查、纯文本预览、复制发布稿、导出 | 现有 ExportModal 和导出工具 |
| 草稿确认流 | 承接 AI 正文和资产建议 | 不自动写入正文或资产 |

## 3. 核心逻辑

- 今日状态区读取现有每日统计、成就连续天数、当前章节自动保存状态、最近快照和备份状态；外部备份失败只做非阻塞提示。
- 本章审稿台使用当前章节正文、角色、伏笔和剧情节点上下文，要求 AI 按固定章节输出报告；若生成结构化资产建议，进入草稿篮。
- 发布前检查先把 HTML 转纯文本，再执行空标题、空正文、敏感词和字数异常检查；复制 / 导出只输出平台中立稿件。
- 资产建议与现有角色库、设定维基、伏笔看板、创世沙盘对齐，不新增平行资产系统。

## 4. 共享层副作用清单

- `ui-store` 增加工作台 / 发布检查入口状态，不改变业务数据语义。
- 新增纯逻辑工具会被工作区状态、审稿台和发布检查 UI 复用。
- 可能读取现有快照、备份、每日统计、角色、伏笔、剧情节点和章节 IPC；不新增外部平台 API。
- 若补充主进程 IPC，只限读取最近备份 / 手动备份 / 导出所需状态，不改 Google Drive 同步语义。

## 5. 验收标准

- AC-DAILY-001：打开有当前章节的作品时，工作区可见今日字数、目标差额、连续日更、保存状态和发布准备入口。
- AC-DAILY-002：无快照、无备份、备份失败、未登录云同步时状态可区分，且不阻塞正文编辑。
- AC-REVIEW-001：本章审稿报告包含六个固定段落：剧情推进、角色一致性、伏笔回收、毒点风险、节奏问题、可执行修改建议。
- AC-REVIEW-002：AI 生成的正文 / 资产改动必须进入草稿确认流，用户拒绝后不写入。
- AC-PUBLISH-001：当前章节 / 全书检查都能列出敏感词、空标题、空正文和字数异常。
- AC-PUBLISH-002：复制发布稿和 TXT / DOCX / Markdown 导出不要求平台账号。
