# Skill 常见反模式

这份文件收口 AI-OS Skill 最常见的失败写法，以及推荐修正方式。

## 1. 触发条件太空

现象：

- `description` 只写“帮助处理 X”
- 看不出什么时候该触发，什么时候不该触发

修正：

- 改成“做什么 + 何时使用”
- 在正文里补 `使用时机` / `触发条件`

## 2. 只有原则，没有步骤

现象：

- 全是理念、规范、审美判断
- 实际执行时不知道先做什么

修正：

- 增加 `使用方式`、`操作步骤` 或 `流程`
- 让 Skill 至少能指导一次真实执行

## 3. 没有边界

现象：

- Skill 只说“能做什么”，不说“什么不做”
- 容易在不适合的上下文里误触发

修正：

- 增加 `约束`、`禁止事项` 或 `适用边界`
- 写清哪些输入缺失时必须先补信息

## 4. `SKILL.md` 变成文档仓库

现象：

- 一份 `SKILL.md` 动辄数百行
- 大量长表格、长说明、长案例都堆在入口文件里

修正：

- 入口文件只保留触发、执行、边界、模板
- 长内容拆到 `references/`

## 5. 只有结论，没有模板

现象：

- 知道要输出什么，但不知道按什么格式输出

修正：

- 增加 `模板引用`、`交付输出` 或 `报告模板`
- 至少给出一个稳定的输出骨架

## 6. 目录名与 `name` 不一致

现象：

- 目录是一个名字，frontmatter 里又是另一个名字

修正：

- 统一命名
- 让目录、`name`、触发描述保持一致

## 7. references 存在但不可导航

现象：

- 建了 `references/`，但没有入口文件
- 读者不知道从哪一篇开始

修正：

- 增加 `references/index.md`
- 把文件按主题归类，而不是无序堆放
