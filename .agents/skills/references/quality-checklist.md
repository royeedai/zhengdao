# Skill 质量清单

用这份清单判断一个 Skill 是否已经达到“可稳定复用”的水平。

## 评分方式

- `2` = 已满足
- `1` = 部分满足
- `0` = 缺失

建议：

- 基线可用：总分至少 `12 / 18`
- 生产级可用：总分至少 `20 / 24`
- 所有“关键项”都不应低于 `2`

## A. 基线可靠性（关键项）

1. frontmatter 存在，`name` 合法
2. `description` 说清了“做什么 + 何时使用”
3. 存在明确的触发条件 / 使用时机
4. 存在明确的执行步骤 / 使用方式
5. 存在边界、约束或禁止事项
6. 标题、命名、术语前后一致

## B. 可交付性

7. 有输出模板、报告模板、交付格式或明确产出
8. 不是只有理念，也不是只有口号
9. 复杂操作有最小示例或模板
10. 高风险动作前有提醒、确认或防误用说明

## C. 可维护性

11. 长内容已拆到 `references/`
12. 若有 `references/`，存在 `references/index.md`
13. 维护信息完整：来源、更新时间、已知限制
14. 脚本和静态资源最小化，不堆空结构

## D. 常见扣分原因

- `description` 很空泛，导致触发条件模糊
- `SKILL.md` 越写越长，最后变成知识堆
- 完全没有边界，任何相似任务都可能误触发
- 写了很多原则，但缺少实际步骤和模板
- 已经存在 `references/`，但没有入口索引
