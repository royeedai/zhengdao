# 用户体验检查清单（详细版）

本文件是 `fullstack-dev-checklist` Skill 的补充参考，提供用户体验相关的详细检查规范。

---

## 管理后台 UX 检查

### 加载与状态
- [ ] **首次加载**：页面打开时有 loading 动画或骨架屏
- [ ] **操作加载**：点击按钮后有 loading 状态（按钮变灰/显示转圈）
- [ ] **空状态**：列表无数据时显示空状态插图 + 友好文案 + 引导操作按钮
- [ ] **错误状态**：网络错误时显示重试提示

### 操作反馈
- [ ] **创建成功**：Toast 提示"创健成功" + 自动跳转回列表
- [ ] **更新成功**：Toast 提示"保存成功" + 留在当前页或跳转
- [ ] **删除确认**：弹出确认弹窗"确定要删除[名称]吗？此操作不可恢复" + 确认/取消按钮
- [ ] **删除成功**：Toast 提示"删除成功" + 刷新列表
- [ ] **批量操作**：显示操作影响的数量"确定要删除选中的 X 项吗？"

### 表单交互
- [ ] **必填提示**：必填字段用红色 `*` 标记
- [ ] **实时校验**：字段失焦时校验并即时显示错误信息
- [ ] **提交校验**：提交时汇总显示所有错误
- [ ] **防重复提交**：提交期间禁用保存按钮
- [ ] **未保存提醒**：离开修改过的表单时提醒"有未保存的修改"
- [ ] **图片上传**：
  - 显示上传进度
  - 支持预览已上传图片
  - 支持删除已上传图片
  - 限制文件大小和格式
  - 拖拽上传（可选）

### 表格交互
- [ ] **行悬停**：鼠标悬停行高亮
- [ ] **列宽合理**：重要信息（名称）列宽大，辅助信息（ID）列宽小
- [ ] **长文本处理**：超长文本截断显示，鼠标悬停显示完整内容（Tooltip）
- [ ] **图片缩略图**：表格中的图片显示缩略图，点击可放大预览
- [ ] **状态标签样式**：不同状态用不同颜色标签（绿色=正常，灰色=禁用，红色=异常）

## 小程序 UX 检查

### 基础体验
- [ ] **下拉刷新**：支持下拉刷新数据
- [ ] **上拉加载**：支持上拉加载更多，有"到底了"提示
- [ ] **页面切换**：Tab 切换流畅，有激活状态标识
- [ ] **返回导航**：每个子页面可以返回上一级
- [ ] **网络异常**：断网时友好提示

### 购物车体验
- [ ] **添加动画**：商品加入购物车时有微动画反馈
- [ ] **数量调整**：`+` 和 `-` 按钮，数量为 1 时 `-` 变为删除确认
- [ ] **库存提示**：超过库存时给出提示
- [ ] **小计更新**：修改数量时金额实时更新

### 订单体验
- [ ] **下单流程**：确认订单 → 选择支付方式 → 支付 → 支付结果
- [ ] **订单状态可视化**：时间轴或进度条展示订单状态流转
- [ ] **倒计时**：待支付订单显示支付倒计时
- [ ] **再来一单**：已完成订单可一键再下单

### 触觉反馈
- [ ] **按钮点击**：有明显的按压效果（hover-class）
- [ ] **滑动操作**：如果有左滑删除等手势操作，有视觉引导

## 通用 UX 原则检查

| 原则 | 检查标准 |
|------|---------|
| 一致性 | 相同功能在不同页面的表现是否一致（如删除操作流程统一） |
| 可见性 | 系统当前状态是否清晰可见（加载中、空状态、错误状态） |
| 反馈性 | 每个用户操作是否有明确的反馈 |
| 容错性 | 危险操作是否有确认机制，是否支持撤销 |
| 效率 | 高频操作是否便捷（如批量操作、快捷搜索） |
| 简约 | 界面是否简洁，不展示无关信息 |
