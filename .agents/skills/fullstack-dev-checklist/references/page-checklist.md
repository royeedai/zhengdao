# 页面级检查清单（详细版）

本文件是 `fullstack-dev-checklist` Skill 的补充参考，提供更细粒度的页面检查项。

---

## 管理后台通用页面模式

### 列表页（List Page）标准模板

```
┌─────────────────────────────────────────────────┐
│ [面包屑导航]                                      │
│                                                   │
│ 页面标题                          [+ 新增] 按钮   │
│                                                   │
│ ┌─搜索/筛选区域──────────────────────────────┐   │
│ │ 关键字搜索  状态筛选  分类筛选  [搜索] [重置] │   │
│ └──────────────────────────────────────────────┘   │
│                                                   │
│ ┌─数据表格───────────────────────────────────┐   │
│ │ □ | ID | 名称 | 分类 | 价格 | 状态 | 操作   │   │
│ │ □ | 1  | 红烧肉| 主食 | ¥32  | 上架 | [编辑]│   │
│ │                                [删除][上/下架]│   │
│ └──────────────────────────────────────────────┘   │
│                                                   │
│ 共 XX 条  [< 1 2 3 ... >]          每页 XX 条    │
└─────────────────────────────────────────────────┘
```

**必须检查：**
1. 表头是否与 .spec 定义的字段一一对应
2. 操作入口是否完整（按钮 / 菜单 / 快捷操作等，参照 .spec 中定义的操作）
3. 全选复选框的批量操作功能
4. 分页组件是否显示总条数
5. 空状态和加载状态
6. 新增 / 编辑流程是否按 benchmark 使用独立页、抽屉或弹窗，而不是把完整表单永久堆在列表同屏

### 表单页（Form Page）标准模板

```
┌─────────────────────────────────────────────────┐
│ [面包屑导航]                                      │
│                                                   │
│ 新增/编辑 XXX                                     │
│                                                   │
│ ┌─表单区域───────────────────────────────────┐   │
│ │ * 名称：    [________________]               │   │
│ │ * 分类：    [请选择分类 ▼    ]  ← 选择器     │   │
│ │   描述：    [________________]               │   │
│ │ * 价格：    [____] 元                        │   │
│ │   图片：    [+ 上传图片]  [预览] [删除]       │   │
│ │ * 状态：    ○ 上架  ○ 下架                   │   │
│ │   排序：    [0___]                           │   │
│ └──────────────────────────────────────────────┘   │
│                                                   │
│                        [取消]  [保存]              │
└─────────────────────────────────────────────────┘
```

**必须检查：**
1. 每个字段的输入组件类型是否合理（参照维度一）
2. 必填字段标记 `*`
3. 编辑模式下数据是否正确回填
4. 取消入口返回列表
5. 提交入口有 loading 状态防重复提交

> 对企业后台类页面，列表页与表单页默认应是分离的交互容器。若采用同屏模式，必须在 spec 中写明 benchmark 依据和适用边界。

## 小程序通用页面模式

### 商品列表/菜单页
- [ ] 分类Tab切换
- [ ] 商品卡片包含：图片、名称、价格、简介
- [ ] 加入购物车按钮/数量选择器
- [ ] 下拉刷新 + 上拉加载更多
- [ ] 搜索功能

### 购物车页
- [ ] 商品列表：图片、名称、单价、数量调整（+/-）、小计
- [ ] 全选/反选
- [ ] 删除已选商品
- [ ] 合计金额实时计算
- [ ] 空购物车提示（带"去选购"引导按钮）
- [ ] 结算按钮（显示已选数量和总价）

### 订单页
- [ ] 订单状态Tab筛选（全部/待支付/进行中/已完成）
- [ ] 订单卡片：订单号、下单时间、商品摘要、总金额、状态标签
- [ ] 操作按钮根据状态动态展示（待支付→去支付/取消，进行中→无，已完成→再来一单）
- [ ] 订单详情页：配送信息、商品明细、支付信息、状态时间线
