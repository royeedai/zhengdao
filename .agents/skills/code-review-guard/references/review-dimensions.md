# 自审维度定义

本文件详细定义 `code-review-guard` Skill 中自审的具体维度和检查标准。

---

## 自审评分标准

### 总体评分规则
- **10分**：所有 .spec 功能完整实现，六维度全部通过，无遗漏
- **8-9分**：核心功能完整，有少量非关键表现细节遗漏
- **6-7分**：主要功能实现，但有明显遗漏（如缺少表单校验、空状态未处理）
- **4-5分**：基本功能存在，但缺失较多交互细节和错误处理
- **1-3分**：核心功能不完整或有严重逻辑错误

---

## 阻塞交付的问题（必须修复）

以下任一问题存在，都**不能标记为完成**：

### P0 - 功能缺失
- .spec 中定义的功能需求未实现
- 缺少关键操作入口或关键能力无法触发（按钮 / 菜单 / 命令 / 手势等）
- 关键能力只有静态展示、占位态或假流程，用户无法完成任务
- 表单缺少必要字段
- API 接口缺失（.spec 定义了但未实现）

### P1 - 逻辑错误
- 关联数据显示 ID 而非名称
- 用户关键任务无法完整走通
- 状态流转逻辑错误
- 前后端字段名不匹配导致数据无法正常传递
- 金额计算逻辑不在后端处理
- 权限检查缺失（未登录可以访问需要登录的接口）

### P2 - 数据安全
- 删除操作没有确认弹窗
- 批量操作没有确认提示
- 敏感数据未脱敏（如用户手机号）
- SQL 注入风险
- 缺少事务处理（涉及多表操作时）

---

## 建议优化的问题（不阻塞但应改进）

### P3 - 体验优化
- 缺少 Loading 状态
- 缺少空数据提示
- 操作成功后无 Toast 提示
- 表单未实现实时校验（仅提交时校验也可接受）
- 缺少面包屑导航

### P4 - 代码质量
- 重复代码未抽取
- 魔法数字/字符串未定义常量
- 错误处理不规范（直接打印日志但未返回错误信息）
- 缺少注释（复杂逻辑处）

### P5 - 性能优化
- 列表查询未分页（一次性加载全部数据）
- 关联查询存在 N+1 问题
- 图片未压缩/未使用缩略图
- 缺少数据库索引

---

## 检查顺序建议

为提高效率，建议按以下顺序检查（先查影响最大的问题）：

1. **功能完整性**（P0）→ 确保 .spec 中的每个 FR 都有对应实现
2. **业务逻辑**（P1）→ 确保逻辑正确、数据展示人性化
3. **数据安全**（P2）→ 确保安全相关的检查到位
4. **用户体验**（P3）→ 确保交互流畅、反馈清晰
5. **代码质量**（P4）→ 确保代码可维护
6. **性能**（P5）→ 确保基本性能合理
