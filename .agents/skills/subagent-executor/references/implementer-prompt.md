# 实现子代理 Prompt 模板

主控 agent 在分配任务时，按此模板构造 prompt。不要让子代理继承主控的会话上下文。

## 模板结构

```markdown
# 任务：[任务名称]

## 项目背景
[2-3 句话描述项目是什么、当前处于什么阶段]

## 设计契约
[从 DESIGN.md / spec 中提取的相关设计决策、接口契约、命名约定]

## 任务详情
[从 tasks.yaml 中提取的任务完整描述，不是摘要]

## 文件清单
- 创建: `exact/path/to/new-file.ts`
- 修改: `exact/path/to/existing.ts`（当前关键内容摘要）
- 测试: `tests/exact/path/to/test.ts`

## 执行要求

1. 严格 TDD：先写失败测试，确认测试失败原因正确，再写最小实现
2. 只修改任务范围内的代码
3. 遵循项目既有模式和命名约定
4. 完成后运行全部相关测试并确认通过

## 交付格式

完成后报告：
- 状态: DONE / DONE_WITH_CONCERNS / NEEDS_CONTEXT / BLOCKED
- 实现摘要: 做了什么
- 测试结果: X/Y 通过
- 自审结论: 是否有需要关注的问题
- （如有顾虑）具体顾虑说明
- （如需上下文）需要什么信息
- （如被阻塞）阻塞原因和建议
```

## 注意事项

- 提供的信息必须自包含，子代理应能仅凭 prompt 完成任务
- 不要只写任务标题让子代理自己去读文件理解
- 相关文件的关键内容应直接包含在 prompt 中
- 如果任务涉及与其他已完成任务的集成点，提供集成上下文
